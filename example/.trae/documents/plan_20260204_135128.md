Based on the analysis of `core_s3` and `m5stack_tab5` BSP, here is the plan to implement `tab5` support following the `core_s3` architecture.

### 1. Modify `wrapper/display.hpp` and `wrapper/display.cpp`
To support MIPI DSI displays (used by Tab5) without creating a separate wrapper class, I will extend the `Display` class to accept pre-initialized IO and Panel handles.
*   **Action**: Add `Init(esp_lcd_panel_io_handle_t io_handle, esp_lcd_panel_handle_t panel_handle, const esp_lcd_panel_dev_config_t& panel_config, CustomLcdPanelInitFunc custom_init_panel_func)` method.
*   **Reason**: `Tab5` uses a complex DSI bus initialization which is best handled in the board file (`tab5.cpp`) rather than complicating the generic `Display` wrapper.

### 2. Create `wrapper-esp32/src/board/m5stack/tab5.hpp`
Define the `M5StackTab5` singleton class, mirroring `M5StackCoreS3`.
*   **Members**:
    *   `I2cBus`: For Touch, Audio, IO Expanders.
    *   `Display`: For the ILI9881C DSI display.
    *   `I2cTouch`: For the GT911 touch controller.
    *   `AudioCodec`: For ES8388 (Speaker) and ES7210 (Mic).
    *   `LvglPort`: For LVGL integration.
    *   `esp_io_expander_handle_t`: For the two PI4IOE5V6408 IO expanders.
*   **Methods**:
    *   `Init()`: Main entry point.
    *   `InitBus()`: Initialize I2C, I2S, and DSI buses.
    *   `InitDevice()`: Initialize IO expanders, Power pins, Display, Touch, Audio.
    *   `InitMiddleware()`: Initialize LVGL.
    *   Getters for all components.
    *   `SetBacklight(uint8_t brightness)`: For backlight control (GPIO 22 PWM).

### 3. Create `wrapper-esp32/src/board/m5stack/tab5.cpp`
Implement the `M5StackTab5` class.
*   **Pin Definitions**: Hardcode pins based on `m5stack_tab5` BSP (I2C: 31/32, I2S: 30/27/29/26/28, BL: 22, etc.).
*   **Implementation Details**:
    *   **IO Expander**: Initialize two `PI4IOE5V6408` instances and enable power for LCD, Touch, Speaker, Camera, USB, WiFi.
    *   **Display**: Initialize DSI bus and ILI9881C panel using ESP-IDF components, then pass handles to `Display` wrapper.
    *   **Touch**: Initialize GT911 using `I2cTouch` wrapper. Handle the special interrupt pin sequence required for GT911.
    *   **Audio**: Initialize `ES8388` and `ES7210` codecs using `AudioCodec` wrapper.
    *   **Backlight**: Implement LEDC PWM control for brightness.

### 4. Dependencies
Ensure `CMakeLists.txt` or `idf_component.yml` includes:
*   `espressif/esp_lcd_ili9881c`
*   `espressif/esp_lcd_touch_gt911`
*   `espressif/esp_io_expander_pi4ioe5v6408`
*   `espressif/esp_codec_dev`
*   `espressif/esp_lvgl_port`

This plan ensures `tab5` is implemented with the same consistent API and structure as `core_s3`, making it easy to use in the `wrapper-esp32` project.
